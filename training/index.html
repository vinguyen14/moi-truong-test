1. Tư duy làm Product: Không phải là viết Code

Chào mừng bạn đến với khóa học. Bài học đầu tiên là bài học quan trọng nhất: Định hình lại tư duy của một người làm sản phẩm.

Nhiều người nghĩ rằng làm ra một trang web SaaS (Software as a Service) thì phải rất giỏi lập trình, phải biết các ngôn ngữ phức tạp. Nhưng thực tế, Code chỉ là công cụ, Sản phẩm sinh ra là để giải quyết Vấn đề.

The Golden Rule

Trước khi viết bất kỳ dòng code nào, hãy tự hỏi: "Nếu không có tính năng này, người dùng có chết không?". Nếu câu trả lời là không, hãy để nó làm sau. Hãy tập trung vào luồng chính (Core value).

Tại sao chúng ta không dùng Framework phức tạp?

Trong khóa học này, chúng ta sẽ xây dựng mọi thứ bằng HTML, Tailwind CSS và Vanilla Javascript kết hợp cùng Google Apps Script (GAS). Vì sao?

  • Chi phí 0 đồng: GAS miễn phí, lưu trữ trên Google Drive miễn phí.
  • Dễ hiểu: Bạn có thể mở file ra, lưu lại và chạy thẳng trên trình duyệt mà không cần cài đặt Node.js hay chạy server.
  • Tốc độ: Nhờ Tailwind CSS, bạn làm giao diện đẹp chuẩn Enterprise chỉ trong vài phút.

2. Kiến trúc Tối giản & Cách làm việc với AI

Để một người không rành công nghệ có thể tự tay làm ra một hệ thống SaaS, bạn cần hai thứ: Một kiến trúc không cần cài đặt, và một người trợ lý AI hiểu việc.

A. Kiến trúc "Zero-Setup" (Không cài đặt)

Thông thường, lập trình viên sẽ bắt bạn cài Node.js, thiết lập Server, mua Database (MySQL). Thật sự rất đau đầu! Trong Masterclass này, chúng ta dùng mô hình tối giản nhất thế giới:

1. Frontend (Giao diện)

Dùng HTML + Tailwind CSS. Mọi thứ được lưu gọn trong 1 file duy nhất, mở bằng Google Chrome là chạy, không cần server.

2. Backend (Database & API)

Sử dụng Google Apps Script (GAS) và Google Sheets. Miễn phí, bảo mật cao, và bạn có thể nhìn thấy dữ liệu của mình bằng mắt thường dưới dạng bảng tính.

B. Công thức Prompt (Ra lệnh) cho AI

Bạn không cần học thuộc code, bạn chỉ cần học cách ra lệnh. Bí quyết để ra lệnh cho AI (như ChatGPT, Claude) viết code đúng ý là: Hãy đóng vai một Tester / Quản lý Dự án trước khi đóng vai Developer.

prompt-template.txt (Công thức chuẩn)
[Role - Đóng vai] Bạn là Senior Fullstack Developer kiêm UI/UX Designer. [Context - Ngữ cảnh] Tôi đang làm tính năng Nộp bài tập cho dự án E-Learning. [Rule - Quy tắc] Chỉ dùng HTML, Tailwind CSS qua CDN. Không dùng React hay thư viện khác. [Task - Nhiệm vụ] Viết giao diện Form gồm: Tên, Link bài tập, và nút "Nộp bài". [Tester Mindset] Nhớ bọc cảnh báo màu đỏ nếu user bấm Nộp mà để trống ô Tên.

Trong bài tập thực hành ở Phase 3, chúng ta sẽ copy nguyên mẫu câu lệnh này để nhờ AI tạo ra toàn bộ giao diện Tool chấm bài tự động!

3. Tailwind CSS: Vẽ giao diện như ráp Lego

Nếu như trước đây, muốn làm một cái nút màu xanh, bạn phải viết 1 file HTML, sau đó tạo 1 file CSS riêng rồi liên kết chúng lại. Với Tailwind, mọi thứ hoàn toàn khác.

Tailwind CSS cung cấp sẵn cho chúng ta hàng ngàn "mảnh ghép Lego" (gọi là Utility classes). Việc của bạn chỉ là chọn từ khóa và gắn thẳng vào thẻ HTML.

Các quy tắc cơ bản dễ nhớ nhất:

  • Màu sắc (Colors): Cú pháp bg-[màu]-[độ đậm] (VD: bg-blue-500, text-red-600).
  • Khoảng cách (Spacing): Chữ p là Padding (lõi bên trong), chữ m là Margin (cách bên ngoài). VD: p-4, mt-2 (margin-top).
  • Bo góc & Đổ bóng: rounded-lg (bo góc tròn), shadow-md (đổ bóng cho nổi khối lên).

Ví dụ thực tế: Thiết kế "Thẻ học viên"

Hãy xem cách chúng ta ráp các class Tailwind lại để tạo ra một cái Card (Thẻ) học viên tuyệt đẹp cực kỳ nhanh chóng.
