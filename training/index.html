1. Tư duy làm Product: Không phải là viết Code

Chào mừng bạn đến với khóa học. Bài học đầu tiên là bài học quan trọng nhất: Định hình lại tư duy của một người làm sản phẩm.

Nhiều người nghĩ rằng làm ra một trang web SaaS (Software as a Service) thì phải rất giỏi lập trình, phải biết các ngôn ngữ phức tạp. Nhưng thực tế, Code chỉ là công cụ, Sản phẩm sinh ra là để giải quyết Vấn đề.

The Golden Rule

Trước khi viết bất kỳ dòng code nào, hãy tự hỏi: "Nếu không có tính năng này, người dùng có chết không?". Nếu câu trả lời là không, hãy để nó làm sau. Hãy tập trung vào luồng chính (Core value).

Tại sao chúng ta không dùng Framework phức tạp?

Trong khóa học này, chúng ta sẽ xây dựng mọi thứ bằng HTML, Tailwind CSS và Vanilla Javascript kết hợp cùng Google Apps Script (GAS). Vì sao?

  • Chi phí 0 đồng: GAS miễn phí, lưu trữ trên Google Drive miễn phí.
  • Dễ hiểu: Bạn có thể mở file ra, lưu lại và chạy thẳng trên trình duyệt mà không cần cài đặt Node.js hay chạy server.
  • Tốc độ: Nhờ Tailwind CSS, bạn làm giao diện đẹp chuẩn Enterprise chỉ trong vài phút.

2. Bí kíp biến AI thành Kỹ sư Trưởng

Bạn không cần phải nhớ code. Nhưng bạn phải biết cách ra lệnh cho máy móc. Nền tảng chúng ta sử dụng là Google AI Studio (với mô hình Gemini Pro Preview) - nơi cho phép bạn "cài đặt nhân cách" cho AI.

Bước 1: Vào phòng điều khiển AI

  • Truy cập aistudio.google.com và đăng nhập bằng Gmail.
  • Bấm nút Create New Prompt (Tạo luồng chat mới).
  • Ở cột bên phải, phần Model, hãy chọn Gemini Pro (Preview) - đây là bộ não thông minh nhất hiện tại.

Chú ý cực kỳ quan trọng

Trong AI Studio có một ô tên là System Instructions (Hướng dẫn hệ thống) nằm ở góc trên/hoặc bên trái. Đây là nơi bạn "nhập hồn" cho AI, bắt nó phải tuân thủ luật lệ của bạn trước khi bắt đầu chat bình thường.

Bước 2: Câu thần chú "System Prompt"

Hãy copy toàn bộ đoạn chữ dưới đây và dán vào ô System Instructions. Đây là bí kíp được dùng để xây dựng các dự án lớn, giúp AI luôn trả về kết quả an toàn, không xóa mất tính năng cũ và luôn giải thích dễ hiểu.

System-Prompt.txt (Dán vào System Instructions)
# VAI TRÒ CỦA BẠN Bạn là một Lập trình viên cấp cao kiêm Người Kiểm duyệt chất lượng (QA/Tester). Bạn giỏi HTML, Tailwind CSS, Javascript và Google Apps Script. Bạn đang hướng dẫn tôi xây dựng trang web. Tôi là người không rành công nghệ, nên hãy giải thích mọi thứ thật đơn giản, không dùng thuật ngữ học thuật. # QUY TẮC TỐI THƯỢNG (KHÔNG ĐƯỢC VI PHẠM) 1. BẢO TOÀN CODE CŨ: Khi tôi nhờ thêm tính năng, phải gắn nó vào code cũ. Tuyệt đối không viết lại file làm mất các tính năng đang chạy. 2. NGUYÊN BẢN NGÔN NGỮ: Giữ nguyên các đoạn chữ tiếng Việt, không tự ý dịch sang tiếng Anh. 3. KHÔNG VIẾT TẮT CODE: Khi xuất code, phải in ra 100% hoàn chỉnh. Nếu sửa 1 đoạn nhỏ, phải chỉ rõ: "Tìm dòng chữ A -> Thay bằng mã B". # QUY TRÌNH LÀM VIỆC (PHẢI TUÂN THỦ 3 BƯỚC) - BƯỚC 1: LẬP KẾ HOẠCH. Đừng vội viết code. Hãy liệt kê ý tưởng ra và hỏi tôi: "[1] Bạn muốn làm giao diện trước? hay [2] Làm dữ liệu trước?". Chờ tôi chọn rồi mới làm. - BƯỚC 2: TỰ KIỂM TRA (TESTER). Trước khi đưa code, hãy tự suy nghĩ xem: "Nếu mạng lag thì sao?", "Nếu tôi nhập sai thì web có sập không?". In suy nghĩ đó ra cho tôi xem. - BƯỚC 3: XUẤT CODE & HƯỚNG DẪN. Cung cấp code và chỉ cho tôi bước tiếp theo cần làm gì. # CHUẨN GIAO TIẾP Luôn dùng Tailwind CSS (gắn thẳng class vào HTML, không cần file CSS riêng). Gọi dữ liệu bằng Fetch API an toàn.

Bước 3: Bắt đầu nhờ AI làm việc

Sau khi cài đặt xong "Nhân vật" ở trên. Xuống ô Chat bình thường và gõ: "Chào bạn, tôi muốn tạo 1 giao diện Form nộp bài tập gồm 2 ô: Tên và Link. Hãy giúp tôi làm Bước 1". Lúc này, bạn sẽ thấy AI làm việc chuyên nghiệp như một Kỹ sư thực thụ!

3. Tailwind CSS: Vẽ giao diện như ráp Lego

Tailwind CSS cung cấp sẵn cho chúng ta hàng ngàn "mảnh ghép Lego". Việc của bạn chỉ là nhờ AI gọi tên mảnh ghép đó và gắn vào khối (thẻ) HTML.

Các quy tắc cơ bản dễ hiểu nhất:

  • Màu sắc (Colors): bg-[màu] là màu nền, text-[màu] là màu chữ.
  • Khoảng cách: Chữ p là lót nệm bên trong, chữ m là đẩy ra bên ngoài. VD: p-4.

4. Tool Chấm Bài (Phần 1): Kho chứa dữ liệu

Chúng ta sẽ bắt đầu dự án thực hành. Để lưu trữ bài tập học viên gửi lên, ta cần một "Cái tủ hồ sơ". Và Google Sheet chính là cái tủ đó.

Bước 1: Lập bảng Google Sheet

Hãy tạo một file Google Sheet mới. Ở dòng số 1, hãy gõ tên các cột sau: id | timestamp | student_name | project_link | status. Đặt tên Sheet bên dưới góc trái là Submissions.

Bước 2: Thuê "Chú bảo vệ" nhận thư (Viết Backend API)

Trên menu Google Sheet, chọn Tiện ích mở rộng > Apps Script. Nó sẽ mở ra một trang trắng. Trang trắng này là nơi ta viết thư cho "Chú bảo vệ" (hàm tên là doPost), dặn chú ấy: "Hễ ai gửi gói dữ liệu có lệnh 'submit', hãy kiểm tra xem có đủ Tên không, nếu đủ thì cất vào Tủ (Sheet) nhé".

Code.gs (Xóa hết code cũ và dán vào)
// Tên hàm bắt buộc của Google: Khi có dữ liệu gửi tới trang web, hàm này sẽ chạy. function doPost(e) { try { // Mở gói hàng người ta gửi tới var data = JSON.parse(e.postData.contents); // Nếu hành động là "submit" (nộp bài) if (data.action === 'submit') { // Nếu thiếu tên hoặc link thì đuổi về, báo lỗi if(!data.student_name || !data.project_link) { return responseJSON("error", "Vui lòng điền đủ Tên và Link bài tập!"); } // Mở cái Tủ (Google Sheet) ra var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("Submissions"); // Tạo 1 mã ngẫu nhiên cho bài nộp (ví dụ: SUB-12345) var newId = "SUB-" + new Date().getTime(); var timeNow = new Date(); // Cất dữ liệu vào Tủ (Thêm 1 dòng mới vào Sheet) sheet.appendRow([newId, timeNow, data.student_name, data.project_link, "Chờ chấm"]); // Báo cáo thành công cho người gửi return responseJSON("success", "Nộp bài thành công!"); } } catch (error) { return responseJSON("error", "Hệ thống bận: " + error.toString()); } } // Hàm phụ: Giúp đóng gói câu trả lời trả về cho đẹp (chuẩn JSON) function responseJSON(status, message) { var output = { "status": status, "message": message }; return ContentService.createTextOutput(JSON.stringify(output)) .setMimeType(ContentService.MimeType.JSON); }

Bước 3: Xuất bản API (Công khai)

Để web có thể gọi được đoạn code này, bạn bấm nút Triển khai (Deploy) > Triển khai mới. Chọn loại là Ứng dụng Web (Web App). Phần Quyền truy cập nhớ chọn Bất kỳ ai (Anyone). Bạn sẽ nhận được 1 đường Link (URL), hãy cất kỹ link này, bài sau ta sẽ dùng nó để ráp vào Giao diện web!
