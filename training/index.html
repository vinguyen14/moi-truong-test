Bước 1: Hiểu Luật Chơi & Vẽ Bản Đồ

1. Nhận diện các loại "Ngôn ngữ"

Bạn không cần tự viết, nhưng khi AI nhả code ra, bạn phải biết nó thuộc loại nào để dán cho đúng.

HTML

Là BỘ KHUNG (tường, cột). Các thẻ luôn có dấu < >. Nó định hình xem web có mấy nút bấm, mấy ô nhập chữ.

CSS (Tailwind)

Là LỚP SƠN. Nó nằm lồng bên trong HTML dưới dạng class="..." để tạo màu sắc, hiệu ứng mờ, bo tròn.

JS / Apps Script

Là ĐỘNG CƠ. Các đoạn code bắt đầu bằng chữ function (hàm xử lý) hoặc const. Giúp web có thể tính toán, lưu dữ liệu.

JSON

Là GÓI HÀNG. Dữ liệu gói lại theo chuẩn {"tên": "An", "điểm": 10} để dễ dàng truyền từ Web sang Google Sheet.

2. Cầu nối API là gì? Tại sao phải dùng?

Giao diện (Web) và Database (Google Sheet) nằm ở 2 nơi khác nhau. API chính là anh "Bồi bàn" kết nối chúng.

Giao diện (Khách)

Học viên nhập thông tin, bấm "Nộp bài".

Gọi món (Fetch API)
Bưng kết quả về

API (Link Google Script)

Tiếp nhận gói hàng (JSON), kiểm tra xem có thiếu Tên không.

Cơ Sở Dữ Liệu (Bếp)

Google Sheet lưu thông tin. Hoặc Google Drive tạo Folder.

3. Lên Kịch bản & Luồng thao tác (User Journey)

Trước khi nhờ AI code, bạn phải gạch đầu dòng được quy trình: Khách hàng làm gì? Mình xử lý gì? Ví dụ về Hệ thống Chấm bài:

Luồng của Học Viên (Nộp)

  • UI: Thấy 1 Form có ô nhập Tên, ô nhập Link File, nút Nộp Bài.
  • Thao tác: Điền form -> Bấm Nút -> Hiện vòng xoay Loading chờ đợi.
  • Hệ thống xử lý: Đóng gói JSON gửi qua API -> Lưu vào Google Sheet.
  • Đầu ra: Hiện thông báo xanh "Đã nộp thành công", tự xóa trắng Form.

Luồng của Giảng Viên (Chấm)

  • UI: Thấy bảng danh sách bài tập. Nhấn làm mới.
  • Thao tác: Bấm nút "Chấm điểm" ở 1 dòng -> Hiện Popup (Cửa sổ).
  • Hệ thống xử lý: Nhập Điểm + Lời phê -> Gửi API (kèm ID bài đó) để ghi đè lên Google Sheet.
  • Đầu ra: Đóng Popup, Bảng tự động đổi màu trạng thái thành "Đã chấm".

3. Khai mở Hệ sinh thái Google (Nâng cao)

Hệ thống API (Apps Script) không chỉ để lưu Google Sheet. Bạn có thể yêu cầu AI viết lệnh điều khiển mọi dịch vụ của Google!

Tạo Folder Tự động

Cơ chế: API nhận tên học viên -> Tìm thư mục gốc -> Tạo thư mục con tên học viên đó -> Trả về link thư mục.

Dùng để: Nộp bài dung lượng lớn, tạo hồ sơ nhân sự.

Xuất PDF / Chứng chỉ

Cơ chế: API copy 1 mẫu Google Docs -> Điền tên/điểm vào chỗ trống -> Convert file đó sang PDF -> Gửi link PDF về Web.

Dùng để: In kết quả, xuất chứng nhận học tập.

Sinh Ảnh Hàng Loạt

Cơ chế: API lấy text chèn vào thư viện vẽ (như Google Slides/Canvas) rồi xuất ra link file Ảnh (.png).

Dùng để: Tạo vé mời sự kiện, thẻ học viên điện tử.

5. Thực chiến: Xây Tool Chấm Bài

Đây là cuốn "Sổ tay lệnh" (Cheat Sheet). Áp dụng đúng 4 bước SOP vừa học, copy các câu lệnh dưới đây giao cho AI, bạn sẽ có ngay sản phẩm.

1. Lệnh tạo Backend (Mở AI Studio & Dán)

Copy đoạn chữ màu xanh bên dưới đưa cho AI để nó viết mã Code.gs cho bạn. Sau đó mang mã dán vào Apps Script và lấy Link API.

Chào Kỹ sư trưởng, tôi đang làm Bước 2 (Backend) cho Tool chấm bài. Vui lòng viết cho tôi mã Google Apps Script (hàm doPost). Bảng Google Sheet của tôi tên là "Submissions", gồm các cột: id, timestamp, student_name, project_link, status, score, feedback. Luồng xử lý: 1. Nếu action = 'submit': Lưu student_name và project_link thành 1 dòng mới, status là "Chờ chấm". 2. Nếu action = 'get_list': Đọc toàn bộ danh sách bỏ qua dòng tiêu đề, trả về JSON. Trả code 100% hoàn chỉnh, nhớ dùng try-catch và xuất kết quả dạng JSON.

2. Lệnh tạo Frontend (Giao diện Web)

Tiếp tục chat với AI câu lệnh này để nó sinh ra file student.html (cho Học viên) và teacher.html (cho Giảng viên). Sau đó tạo file trên GitHub và dán vào.

Tuyệt vời. Bây giờ chuyển sang Bước 3 (Frontend). Viết cho tôi 2 file HTML độc lập dùng Tailwind CSS (gắn qua CDN): File 1 (student.html): Form nộp bài đẹp mắt. Gồm ô Tên, ô Link và nút "Nộp Bài". Bấm nút thì gọi Fetch API gửi dữ liệu action='submit'. Có nút loading khi chờ. File 2 (teacher.html): Dashboard quản trị. Vừa mở web tự động Fetch API gọi action='get_list' để hiển thị danh sách bài nộp ra dạng Bảng (Table). Lưu ý: Header fetch API phải dùng 'Content-Type': 'text/plain'. Code để sẵn 1 biến const API_URL = "" ở đầu thẻ script để tôi tự điền link sau.
