1. Bạn là "Kiến Trúc Sư", không phải thợ xây

Hãy quên đi những đoạn code phức tạp. Để tạo ra một phần mềm web, bạn chỉ cần 3 tư duy cốt lõi sau đây:

Giải quyết nỗi đau

Người dùng không quan tâm web của bạn làm bằng ngôn ngữ gì. Họ chỉ quan tâm: "Nó có giúp tôi làm việc nhanh hơn không?"

Thuê AI làm "thợ"

Bạn có ý tưởng, bạn mô tả nó bằng tiếng Việt. Trí tuệ nhân tạo (AI) sẽ là người kỹ sư cặm cụi gõ từng dòng code cho bạn.

Cắt bỏ sự rườm rà

Đừng vẽ vời tính năng. Hãy làm ra tính năng quan trọng nhất trước. Ví dụ: Làm web chấm bài thì chỉ cần Form Nộp và Form Chấm là đủ.

Quy Tắc Tối Giản của Masterclass

Trong khóa học này, ta KHÔNG mua Server phức tạp. Ta sẽ dùng Trình duyệt web làm màn hình giao diện, và dùng Google Sheet làm kho chứa dữ liệu. Hoàn toàn miễn phí 0 đồng!

Thay vì phải tự pha màu vất vả, Tailwind cung cấp cho bạn hàng ngàn "Từ khóa ma thuật" (Class) đã làm sẵn. Bạn chỉ việc gọi tên, giao diện sẽ đẹp ngay lập tức.

4. Tool Chấm Bài (Phần 1): Kho chứa dữ liệu

Chúng ta sẽ bắt đầu dự án thực hành. Để lưu trữ bài tập học viên gửi lên, ta cần một "Cái tủ hồ sơ". Và Google Sheet chính là cái tủ đó.

Bước 1: Lập bảng Google Sheet

Hãy tạo một file Google Sheet mới. Ở dòng số 1, hãy gõ tên các cột sau: id | timestamp | student_name | project_link | status. Đặt tên Sheet bên dưới góc trái là Submissions.

Bước 2: Thuê "Chú bảo vệ" nhận thư (Viết Backend API)

Trên menu Google Sheet, chọn Tiện ích mở rộng > Apps Script. Nó sẽ mở ra một trang trắng. Trang trắng này là nơi ta viết thư cho "Chú bảo vệ" (hàm tên là doPost), dặn chú ấy: "Hễ ai gửi gói dữ liệu có lệnh 'submit', hãy kiểm tra xem có đủ Tên không, nếu đủ thì cất vào Tủ (Sheet) nhé".

Code.gs (Xóa hết code cũ và dán vào)
// Tên hàm bắt buộc của Google: Khi có dữ liệu gửi tới trang web, hàm này sẽ chạy. function doPost(e) { try { // Mở gói hàng người ta gửi tới var data = JSON.parse(e.postData.contents); // Nếu hành động là "submit" (nộp bài) if (data.action === 'submit') { // Nếu thiếu tên hoặc link thì đuổi về, báo lỗi if(!data.student_name || !data.project_link) { return responseJSON("error", "Vui lòng điền đủ Tên và Link bài tập!"); } // Mở cái Tủ (Google Sheet) ra var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("Submissions"); // Tạo 1 mã ngẫu nhiên cho bài nộp (ví dụ: SUB-12345) var newId = "SUB-" + new Date().getTime(); var timeNow = new Date(); // Cất dữ liệu vào Tủ (Thêm 1 dòng mới vào Sheet) sheet.appendRow([newId, timeNow, data.student_name, data.project_link, "Chờ chấm"]); // Báo cáo thành công cho người gửi return responseJSON("success", "Nộp bài thành công!"); } } catch (error) { return responseJSON("error", "Hệ thống bận: " + error.toString()); } } // Hàm phụ: Giúp đóng gói câu trả lời trả về cho đẹp (chuẩn JSON) function responseJSON(status, message) { var output = { "status": status, "message": message }; return ContentService.createTextOutput(JSON.stringify(output)) .setMimeType(ContentService.MimeType.JSON); }

Bước 3: Xuất bản API (Công khai)

Để web có thể gọi được đoạn code này, bạn bấm nút Triển khai (Deploy) > Triển khai mới. Chọn loại là Ứng dụng Web (Web App). Phần Quyền truy cập nhớ chọn Bất kỳ ai (Anyone). Bạn sẽ nhận được 1 đường Link (URL), hãy cất kỹ link này, bài sau ta sẽ dùng nó để ráp vào Giao diện web!

5. Tool Chấm Bài (Phần 2): Quyền lực của Giảng viên

Ở Bài 4, học viên đã có thể nộp bài vào Kho (Google Sheet). Bây giờ, chúng ta sẽ mở rộng khả năng của "Chú bảo vệ" (API) để phục vụ Giảng viên: Lấy danh sách bài tập về xem, và Chấm điểm.

Nâng cấp "Cái Tủ Hồ Sơ"

Mở file Google Sheet Submissions của bạn ra. Hãy thêm 2 cột nữa vào cạnh cột status: Đó là cột score (Điểm) và cột feedback (Lời phê).

Bổ sung Lệnh cho Chú bảo vệ (Apps Script)

Quay lại trang viết code ban nãy. Chúng ta sẽ nối thêm tính năng vào phía dưới đoạn code của Bài 4. Cụ thể, ta thêm lệnh get_list và grade.

Code.gs (Bản nâng cấp Hoàn chỉnh)
function doPost(e) { try { var data = JSON.parse(e.postData.contents); var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("Submissions"); // 1. TÍNH NĂNG NỘP BÀI (HỌC VIÊN) - Như Bài 4 if (data.action === 'submit') { if(!data.student_name || !data.project_link) return responseJSON("error", "Thiếu thông tin!"); var newId = "SUB-" + new Date().getTime(); // Các cột: id | timestamp | student_name | project_link | status | score | feedback sheet.appendRow([newId, new Date(), data.student_name, data.project_link, "Chờ chấm", "", ""]); return responseJSON("success", "Nộp bài thành công!"); } // 2. TÍNH NĂNG LẤY DANH SÁCH (GIẢNG VIÊN) else if (data.action === 'get_list') { // Lấy toàn bộ dữ liệu trong bảng var rows = sheet.getDataRange().getValues(); var result = []; // Bỏ qua dòng số 1 (tiêu đề), bắt đầu đọc từ dòng 2 (i=1) for (var i = 1; i < rows.length; i++) { result.push({ id: rows[i][0], timestamp: rows[i][1], student_name: rows[i][2], project_link: rows[i][3], status: rows[i][4], score: rows[i][5], feedback: rows[i][6] }); } // Gửi danh sách về cho trang web hiển thị return responseJSON("success", "Lấy dữ liệu thành công", result); } // 3. TÍNH NĂNG CHẤM ĐIỂM (GIẢNG VIÊN) else if (data.action === 'grade') { var rows = sheet.getDataRange().getValues(); // Tìm xem mã bài tập nằm ở dòng số mấy for (var i = 1; i < rows.length; i++) { if (rows[i][0] === data.id) { // Cập nhật cột Trạng thái (cột 5), Điểm (cột 6), Lời phê (cột 7) sheet.getRange(i + 1, 5).setValue("Đã chấm"); sheet.getRange(i + 1, 6).setValue(data.score); sheet.getRange(i + 1, 7).setValue(data.feedback); return responseJSON("success", "Chấm điểm thành công!"); } } return responseJSON("error", "Không tìm thấy bài tập này!"); } } catch (error) { return responseJSON("error", "Lỗi: " + error.toString()); } } // Hàm phụ: Cập nhật hàm này để có thể gửi kèm dữ liệu (data) về web function responseJSON(status, message, data = null) { var output = { "status": status, "message": message }; if (data) output.data = data; return ContentService.createTextOutput(JSON.stringify(output)) .setMimeType(ContentService.MimeType.JSON); }

Cập nhật Phiên bản (Deploy)

Mỗi lần sửa code, bạn phải tạo phiên bản mới. Bấm Deploy > Manage Deployments (Quản lý bản triển khai), bấm vào hình cây bút (Edit), ở chỗ Version chọn New version và bấm Deploy. Link URL cũ vẫn giữ nguyên không đổi!

6. Xây Dựng Giao Diện Nộp Bài

Chúng ta đã có "Tủ hồ sơ" (Google Sheet) và "Chú bảo vệ" (API). Bây giờ, ta cần làm một cái Bưu điện (Giao diện Web) để học viên đến gửi thư.

Bước 1: Tạo Form nhập liệu (HTML)

Hãy mở máy tính của bạn, tạo một file tên là student.html. Dùng câu thần chú ở Bài 2, yêu cầu AI viết cho bạn một cái Form nhập Tên và Link bằng Tailwind CSS. Ta sẽ có đoạn mã dạng như sau:

student.html (Phần Giao diện)
<input id="studentName" placeholder="Nhập tên của bạn" class="border p-2"> <input id="projectLink" placeholder="Link bài tập" class="border p-2"> <button onclick="submitData()" class="bg-blue-600 text-white"> Nộp bài </button>

Bước 2: Thuê "Anh Shipper" (Lập trình JavaScript)

Khi học viên bấm nút "Nộp bài", dữ liệu vẫn nằm trên máy tính của họ. Ta cần viết một đoạn mã JavaScript (Anh Shipper) để lấy dữ liệu đó và chạy tới địa chỉ API của ta. Trong lập trình, "Anh Shipper" này tên là fetch().

Quy tắc Vàng chống Lỗi CORS

Rất nhiều bạn mới làm bị lỗi trình duyệt chặn gửi dữ liệu (CORS Error). Bí quyết là luôn phải bọc gói hàng bằng giấy gói text/plain. Hãy xem đoạn mã bên dưới.

Thêm thẻ <script> vào dưới cùng file student.html
// 1. Dán đường link API (Web App) lấy từ Bài 4 vào đây const API_URL = "https://script.google.com/macros/s/.../exec"; function submitData() { // 2. Lấy dữ liệu người dùng gõ vào ô nhập const name = document.getElementById('studentName').value; const link = document.getElementById('projectLink').value; // 3. Đóng gói hàng hóa const payload = { action: "submit", student_name: name, project_link: link }; // 4. Gọi Shipper (fetch) mang đi giao fetch(API_URL, { method: 'POST', headers: { 'Content-Type': 'text/plain;charset=utf-8' }, // Bí kíp chống CORS body: JSON.stringify(payload) }) .then(response => response.json()) .then(data => { alert(data.message); // Báo thành công! }); }

Chúc mừng! Bạn đã hoàn thành một chu trình Fullstack hoàn chỉnh. Hãy thử mở file student.html của bạn lên, nhập dữ liệu và xem điều kỳ diệu xảy ra trong Google Sheet nhé!

7. Trạm Chấm Bài (Teacher Dashboard)

Mảnh ghép cuối cùng để hoàn thiện một Hệ thống SaaS thực thụ.

Khác với học viên chỉ có chức năng Gửi dữ liệu đi. Ở giao diện Giảng viên, chúng ta cần làm 2 việc: Lấy danh sách về (Đọc) và Chấm điểm (Ghi đè).

1. Lấy dữ liệu về Bảng (Table)

Ta sẽ tạo một file teacher.html. Dùng AI thiết kế một cái Bảng danh sách. Sau đó, nhờ "Anh Shipper" (hàm fetch) mang theo lời nhắn action: 'get_list' chạy lên Google Sheet gom hết bài tập mang về.

Lệnh lấy danh sách bài nộp
fetch(API_URL, { method: 'POST', headers: { 'Content-Type': 'text/plain' }, body: JSON.stringify({ action: 'get_list' }) // Báo API biết là tôi muốn lấy danh sách }) .then(res => res.json()) .then(response => { // Nhận được danh sách, dùng Javascript vẽ nó ra thành các hàng <tr> <td> trên web console.log("Đã lấy được: ", response.data); });

2. Chấm điểm & Trả kết quả

Khi giảng viên nhập điểm 10 và bấm Lưu. Ta lại đóng gói một kiện hàng mới, gồm 3 thông tin quan trọng: Mã Bài Tập (ID), Điểm, và Lời phê. Bắn thẳng lên API.

Lệnh chấm điểm cập nhật Google Sheet
const payload = { action: "grade", id: "SUB-12345", // Để API biết đang chấm bài của ai score: "10", feedback: "Làm rất tốt!" }; fetch(API_URL, { method: 'POST', headers: { 'Content-Type': 'text/plain' }, body: JSON.stringify(payload) }) .then(res => res.json()) .then(data => alert("Đã lưu điểm thành công!"));
