1. Bạn là "Kiến Trúc Sư", không phải thợ xây

Hãy quên đi những đoạn code phức tạp. Để tạo ra một phần mềm web, bạn chỉ cần 3 tư duy cốt lõi sau đây:

Giải quyết nỗi đau

Người dùng không quan tâm web của bạn làm bằng ngôn ngữ gì. Họ chỉ quan tâm: "Nó có giúp tôi làm việc nhanh hơn không?"

Thuê AI làm "thợ"

Bạn có ý tưởng, bạn mô tả nó bằng tiếng Việt. Trí tuệ nhân tạo (AI) sẽ là người kỹ sư cặm cụi gõ từng dòng code cho bạn.

Cắt bỏ sự rườm rà

Đừng vẽ vời tính năng. Hãy làm ra tính năng quan trọng nhất trước. Ví dụ: Làm web chấm bài thì chỉ cần Form Nộp và Form Chấm là đủ.

Quy Tắc Tối Giản của Masterclass

Trong khóa học này, ta KHÔNG mua Server phức tạp. Ta sẽ dùng Trình duyệt web làm màn hình giao diện, và dùng Google Sheet làm kho chứa dữ liệu. Hoàn toàn miễn phí 0 đồng!

3. Vẽ web đẹp như ráp Lego

Hãy tưởng tượng: HTML là những viên gạch xây nên khung nhà thô. Còn Tailwind chính là đội Thợ Sơn và Thợ Nội Thất trang trí cho ngôi nhà đó.

Thay vì phải tự pha màu vất vả, Tailwind cung cấp cho bạn hàng ngàn "Từ khóa ma thuật" (Class) đã làm sẵn. Bạn chỉ việc gọi tên, giao diện sẽ đẹp ngay lập tức.

1. Đổ màu sắc

Dùng chữ bg (Nền) hoặc text (Chữ) + Tên màu.

bg-blue-500
text-red-600 Chữ đỏ

2. Bơm khoảng trống

Dùng chữ p (Lót nệm bên trong) hoặc m (Đẩy ra bên ngoài).

p-4 = Nệm dày mức 4
mt-2 = Cách phía trên mức 2

3. Bo góc & Đổ bóng

Gọt dũa cho khối hình vuông vức trở nên mềm mại.

rounded-xl
shadow-lg

Ví dụ: Cách ráp lệnh tạo ra "Thẻ học viên"

Bạn KHÔNG CẦN phải học thuộc lòng các "từ khóa" này. Ở các bước sau, AI sẽ tự động gõ mã Code đó cho bạn. Bạn chỉ cần học cách "Đọc hiểu" để có thể tự sửa (Ví dụ: Thấy chữ màu đỏ chưa đẹp, bạn tìm từ khóa text-red-500 đổi thành text-blue-500 là xong!).

4. Tủ Hồ Sơ & Chú Bảo Vệ

Bước đầu tiên của dự án: Tạo một nơi để cất giữ bài tập của học viên.

1. "Tủ Hồ Sơ" (Google Sheet)

Mở Google Sheet tạo 1 file mới. Ở dòng đầu tiên gõ các cột: id, timestamp, student_name, project_link, status. Đặt tên Sheet là Submissions.

2. "Chú Bảo Vệ" (Apps Script)

Tủ hồ sơ không thể để ai cũng mở được. Chọn Tiện ích mở rộng > Apps Script. Đây là nơi ta viết luật cho chú bảo vệ.

Nhiệm vụ 1: Nhận thư & Cất vào tủ

Dán đoạn mã (luật) sau vào trang Apps Script. Hàm doPost chính là chú bảo vệ. Nếu có ai gửi gói hàng (submit) có đủ Tên và Link, chú sẽ cất nó thành 1 dòng mới trong Tủ Hồ Sơ.

Mã lệnh cho Chú Bảo Vệ (Code.gs)
function doPost(e) { try { var data = JSON.parse(e.postData.contents); var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("Submissions"); // Nếu nhận được lệnh 'submit' (Nộp bài) if (data.action === 'submit') { var newId = "SUB-" + new Date().getTime(); // Cất vào Tủ (Thêm 1 dòng vào Sheet) sheet.appendRow([newId, new Date(), data.student_name, data.project_link, "Chờ chấm"]); // Báo cáo thành công! return responseJSON("success", "Nộp bài thành công!"); } } catch (error) { ... } }

Cấp địa chỉ cho Bảo Vệ (Deploy API)

Bấm Triển khai (Deploy) > Web App > Quyền: Bất kỳ ai. Bạn sẽ nhận được 1 đường Link (URL). Đây chính là "địa chỉ bưu điện" để lát nữa ta gọi Shipper mang dữ liệu tới!

Bổ sung cột Điểm số và dạy cho "Chú bảo vệ" 2 kỹ năng mới phục vụ Giảng Viên.

6. Thuê "Anh Shipper" giao hàng

Tạo trang Web student.html bằng Tailwind. Sau đó dùng Fetch API (Anh Shipper) để gửi bài nộp đi.

Người dùng gõ Tên và Link lên Form. Bấm nút "Nộp bài", JavaScript sẽ gói gọn nó lại thành một kiện hàng (Payload) và gọi Shipper mang lên Địa chỉ API của bạn.

Mã lệnh JavaScript (Cách gửi hàng)
// 1. Đóng gói hàng hóa (Payload) const payload = { action: "submit", student_name: "Trần Văn A", project_link: "https://..." }; // 2. Gọi Shipper (Fetch) mang đi fetch(API_URL, { method: 'POST', headers: { 'Content-Type': 'text/plain;charset=utf-8' }, // <-- Giấy thông hành chống chặn body: JSON.stringify(payload) }) .then(data => alert("Giao hàng thành công!"));

Thẻ Thông Hành Chống Lỗi (CORS)

Trình duyệt rất hay chặn việc gửi dữ liệu lung tung. Bí quyết của Masterclass là luôn dán nhãn text/plain lên gói hàng. Google sẽ vui vẻ mở cửa cho đi qua cái vèo!

7. Trạm Chỉ Huy (Web Giảng Viên)

Mảnh ghép cuối cùng: Tạo Dashboard teacher.html để hiển thị Bảng danh sách và chấm điểm.

Bảng điều khiển này hoạt động theo 2 luồng liên tục: Vừa mở lên là Nhận danh sách về (Đọc), bấm nút Lưu là Gửi điểm đi (Ghi).

Lấy Danh Sách

body: JSON.stringify({ action: 'get_list' })

Dùng JavaScript vẽ kết quả trả về thành các hàng ngang <tr> trên Bảng hiển thị.

Gửi Điểm Chấm

body: JSON.stringify({ action: "grade", id: "SUB-123", score: "10" })

API sẽ dựa vào ID để tìm đúng bài tập trong Google Sheet và cập nhật điểm.
