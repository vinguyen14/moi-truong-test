1. Cấu trúc Website & Quản trị Code bằng GitHub

Trước khi dùng AI viết code, bạn phải hiểu cốt lõi thư mục của một trang web và cách lưu trữ nó an toàn.

1. Các thành phần gốc của Website

index.html (Trang chủ)

Đây là "khung xương" chính. Khi mở website, trình duyệt luôn đọc file này đầu tiên. Toàn bộ Layout chính, Sidebar, Header đều nằm ở đây.

*.html (File con)

Ví dụ `student.html`. Để nhúng nó vào file lớn `index.html`, ta dùng thẻ link (Ví dụ: `<a href="student.html">`), bấm vào sẽ mở tab mới.

*.json (Dữ liệu)

Đây không phải giao diện, mà là file chứa dữ liệu thô (ví dụ: danh sách bài tập, thông tin user) để code JS đọc và vẽ ra màn hình.

2. Quản lý Source Code trên GitHub

Mọi dự án chuyên nghiệp đều phải lưu trên GitHub (môi trường đám mây). Các bước vận hành chuẩn như sau:

  • 1
    Tạo Repository (Kho chứa): Đăng nhập GitHub, bấm New Repository. Nhập tên dự án (VD: trainer-os). Để chế độ Public hoặc Private.
  • 2
    Add File & Tạo Folder: Trong Repo, bấm Add file > Create new file.
    👉 Mẹo tạo Folder: Gõ tên thư mục, sau đó gõ dấu gạch chéo / (VD: pages/), GitHub sẽ tự biến nó thành Folder.
    👉 Lưu File: Tên file BẮT BUỘC phải có đuôi mở rộng, ví dụ index.html hoặc app.js. Nếu lưu không đuôi, code sẽ không bao giờ chạy.
  • 3
    Nhận biết trạng thái Lưu Code (VS Code / Web):
    Chấm Xanh lá (U - Untracked/Added): File mới tạo, đã sẵn sàng lưu lên hệ thống.
    Chấm Cam/Nâu (M - Modified): File cũ vừa bị chỉnh sửa nhưng CHƯA ĐƯỢC SAVE. Bạn phải bấm Commit (Lưu) thì code mới chạy bản mới nhất.

Quy tắc 2 Môi trường (Best Practice)

Tuyệt đối không sửa code thẳng trên trang đang chạy thực tế. Luôn tạo 2 Repo: [Tên-Dự-án]-Test và [Tên-Dự-án]-Prod. Bạn nhờ AI viết code, ráp vào bản Test. Chạy thử mọi ngóc ngách, nếu ngon lành mới Copy file đó đè sang bản Prod (Thực tế) cho user dùng.

2. Trạm Điều Khiển AI & Bộ Prompt Quyền Lực

Setup AI chuẩn xác quyết định 90% sự thành bại của dự án. Giao sai model hoặc thiếu luật, code sẽ rối tung.

1. Lựa chọn Model AI (Google AI Studio)

Mở aistudio.google.com. Bắt buộc chọn Gemini 1.5 Pro (hoặc bản thử nghiệm Gemini 3.1 Pro Preview nếu có).

Tại sao phải dùng bản Pro?

Các dự án thực tế file code dài cả ngàn dòng. Các bản Flash hay AI rẻ tiền đọc sẽ bị "quên" logic. Bản Pro có cửa sổ ngữ cảnh cực lớn (Context window), đọc hiểu toàn bộ project không trượt hàm nào. (Ghi chú: Nếu hệ thống yêu cầu tài khoản trả phí, hãy đăng nhập bằng tài khoản nội bộ Citigold để xử lý code hiệu quả nhất).

2. Tư duy Kế thừa Dự án (Google Docs)

Khi muốn tạo ra một tính năng mới hoặc một Web mới. Đừng yêu cầu AI bắt đầu từ số 0.

  • Hãy copy toàn bộ mã index.html và Code.gs của phiên bản hoàn chỉnh trước đó (Ví dụ: Trainer OS).
  • Dán chúng vào 1 file Google Docs, xuất ra định dạng PDF hoặc TXT.
  • Upload file đó lên AI Studio và ra lệnh: "Đây là sườn kiến trúc cũ của tôi, hãy lấy nó làm gốc để phát triển thêm giao diện X". Code mới sinh ra sẽ tự động khớp 100% style của bạn.

3. Bộ Prompt Kỷ Luật Hệ Thống (Dán vào System Instructions)

Copy toàn bộ khung dưới đây dán vào mục System Instructions. Đây là bộ luật tối cao ép AI phải tuân thủ luồng công việc 3 bước và giữ nguyên cấu trúc dự án của bạn.

BẢO TOÀN NỘI DUNG VĂN BẢN (TEXT/CONTENT): Tuyệt đối không thay đổi nội dung tiếng Việt, các đoạn text hardcode trong `<textarea>`, placeholder, alert trừ khi user yêu cầu. 3. KHÔNG DÙNG PLACEHOLDER TRONG CODE: Khi yêu cầu cung cấp "Toàn bộ file", phải in ra 100% code hoàn chỉnh. Dừng lại CHỜ USER XÁC NHẬN. - BƯỚC 2 - MÔ PHỎNG TESTER (SELF-TESTING): In ra suy nghĩ rà soát chéo các hàm JS, biến, CSS hiện tại có bị xung đột không. - BƯỚC 3 - XUẤT CODE & HƯỚNG DẪN: Xuất code và gợi ý bước tiếp theo. # ⚙️ ARCHITECTURE & LOGIC STANDARDS (CHUẨN CODE CỦA DỰ ÁN) 1. ĐỒNG BỘ CHÉO (CROSS-FILE SYNC): Giao diện phải ĐỒNG BỘ 100%. Bám sát các class đã có như `.saas-card`, `.btn-primary`, sidebar layout. 2. CHUẨN GIAO TIẾP API: BẮT BUỘC dùng: `method: 'POST', headers: { 'Content-Type': 'text/plain;charset=utf-8' }, body: JSON.stringify(payload)`. Luôn xử lý `.then(r => r.json())`.

3. Kỹ năng Cắt Ghép Code & Tối Ưu Token

Tránh việc bắt AI viết lại toàn bộ file dài 1000 dòng. Vừa tốn Token, vừa làm AI bị "đứt gánh" giữa chừng.

1. Kỹ thuật thay thế từng đoạn (Ctrl + F)

Thay vì copy cả file nhờ sửa. Hãy chỉ copy đoạn hàm/thẻ HTML bị lỗi. Sau khi AI trả code về, làm theo các bước:

Tìm hàm JavaScript

Bấm Ctrl + F (hoặc Cmd + F) trong GitHub/VS Code. Gõ tên hàm, ví dụ: function submitData(). Bôi đen từ chữ function đến dấu ngoặc nhọn đóng } của hàm đó và dán code mới đè lên.

Tìm thẻ HTML lắt nhắt

Nếu muốn sửa 1 dòng text (VD: đổi chữ "Nộp bài"). Bấm Ctrl + F gõ chính xác chữ "Nộp bài", hoặc gõ ID của nút đó id="btnSubmit". Thay đúng dòng đó, không đụng vào layout xung quanh.

2. Tại sao add code vào nhưng Web không chạy?

Khi gặp lỗi trắng trang hoặc không có gì thay đổi, hãy rà soát 4 nguyên nhân gốc rễ sau:

  • Lỗi chưa Save (Chấm Cam/M): Đã dán code vào GitHub/Editor nhưng quên bấm Commit/Save. Trang web vẫn đọc bản cũ.
  • Lỗi Cache trình duyệt: Code mới đã lưu, nhưng trình duyệt vẫn lưu bản cũ cho nhẹ máy. Bấm Ctrl + F5 (hoặc Cmd + Shift + R) để tải lại toàn bộ mã nguồn sạch.
  • Thiếu đóng thẻ HTML/JS: Khi dán đè code, bạn vô tình làm mất 1 dấu ngoặc nhọn } ở cuối hàm JS, hoặc mất thẻ </div>. Lỗi này làm chết toàn bộ luồng chạy của file.
  • Thiếu đuôi .html: File tạo mới ghi là index thay vì index.html, máy tính sẽ không hiểu đây là trang web.

4. Tủ Hồ Sơ & Chú Bảo Vệ

Bước đầu tiên của dự án: Tạo một nơi để cất giữ bài tập của học viên.

1. "Tủ Hồ Sơ" (Google Sheet)

Mở Google Sheet tạo 1 file mới. Ở dòng đầu tiên gõ các cột: id, timestamp, student_name, project_link, status. Đặt tên Sheet là Submissions.

2. "Chú Bảo Vệ" (Apps Script)

Tủ hồ sơ không thể để ai cũng mở được. Chọn Tiện ích mở rộng > Apps Script. Đây là nơi ta viết luật cho chú bảo vệ.

Nhiệm vụ 1: Nhận thư & Cất vào tủ

Dán đoạn mã (luật) sau vào trang Apps Script. Hàm doPost chính là chú bảo vệ. Nếu có ai gửi gói hàng (submit) có đủ Tên và Link, chú sẽ cất nó thành 1 dòng mới trong Tủ Hồ Sơ.

Mã lệnh cho Chú Bảo Vệ (Code.gs)
function doPost(e) { try { var data = JSON.parse(e.postData.contents); var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("Submissions"); // Nếu nhận được lệnh 'submit' (Nộp bài) if (data.action === 'submit') { var newId = "SUB-" + new Date().getTime(); // Cất vào Tủ (Thêm 1 dòng vào Sheet) sheet.appendRow([newId, new Date(), data.student_name, data.project_link, "Chờ chấm"]); // Báo cáo thành công! return responseJSON("success", "Nộp bài thành công!"); } } catch (error) { ... } }

Cấp địa chỉ cho Bảo Vệ (Deploy API)

Bấm Triển khai (Deploy) > Web App > Quyền: Bất kỳ ai. Bạn sẽ nhận được 1 đường Link (URL). Đây chính là "địa chỉ bưu điện" để lát nữa ta gọi Shipper mang dữ liệu tới!

6. Thuê "Anh Shipper" giao hàng

Tạo trang Web student.html bằng Tailwind. Sau đó dùng Fetch API (Anh Shipper) để gửi bài nộp đi.

Người dùng gõ Tên và Link lên Form. Bấm nút "Nộp bài", JavaScript sẽ gói gọn nó lại thành một kiện hàng (Payload) và gọi Shipper mang lên Địa chỉ API của bạn.

Mã lệnh JavaScript (Cách gửi hàng)
// 1. Đóng gói hàng hóa (Payload) const payload = { action: "submit", student_name: "Trần Văn A", project_link: "https://..." }; // 2. Gọi Shipper (Fetch) mang đi fetch(API_URL, { method: 'POST', headers: { 'Content-Type': 'text/plain;charset=utf-8' }, // <-- Giấy thông hành chống chặn body: JSON.stringify(payload) }) .then(data => alert("Giao hàng thành công!"));

Thẻ Thông Hành Chống Lỗi (CORS)

Trình duyệt rất hay chặn việc gửi dữ liệu lung tung. Bí quyết của Masterclass là luôn dán nhãn text/plain lên gói hàng. Google sẽ vui vẻ mở cửa cho đi qua cái vèo!

7. Trạm Chỉ Huy (Web Giảng Viên)

Mảnh ghép cuối cùng: Tạo Dashboard teacher.html để hiển thị Bảng danh sách và chấm điểm.

Bảng điều khiển này hoạt động theo 2 luồng liên tục: Vừa mở lên là Nhận danh sách về (Đọc), bấm nút Lưu là Gửi điểm đi (Ghi).

Lấy Danh Sách

body: JSON.stringify({ action: 'get_list' })

Dùng JavaScript vẽ kết quả trả về thành các hàng ngang <tr> trên Bảng hiển thị.

Gửi Điểm Chấm

body: JSON.stringify({ action: "grade", id: "SUB-123", score: "10" })

API sẽ dựa vào ID để tìm đúng bài tập trong Google Sheet và cập nhật điểm.
